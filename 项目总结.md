# 武汉操作票信息抽取项目总结

## 项目概述

本项目基于数据库中的人工标注数据，学习并实现了操作票文本的自动信息抽取功能。通过分析3894条CCZL类型的标注数据，开发了能够自动从操作票文本中抽取6个关键字段的系统。

## 数据源信息

**数据库配置:**
- 主机: 192.168.64.58:3306
- 用户: root
- 数据库: operate_ticket

**数据表:**
- `t_annotation_val`: 包含标注值数据（6个目标字段）
- `t_text_annotation_info`: 包含原始文本数据
- 筛选条件: `text_id` 和 `id` 以 "CCZL" 结尾的数据

**数据规模:**
- 总计提取了 3894 条 CCZL 标注数据
- 用于模式分析和算法训练

## 目标字段

需要从操作票文本中抽取的6个字段：

1. **初始状态** (`initial_action`) - 设备的起始状态
2. **目标状态** (`target_action`) - 设备的目标状态  
3. **操作** (`device_location`) - 执行的操作类型
4. **操作设备** (`device_name`) - 被操作的设备名称
5. **设备所属厂站** (`supply_target_device_name`) - 设备所属的厂站
6. **设备所属线路** (`initial_connect_bus_name`) - 设备所属的线路

## 项目实施过程

### 1. 数据提取阶段
- 连接MySQL数据库，查看表结构
- 提取所有CCZL结尾的标注数据和对应文本
- 数据清洗和格式化处理

### 2. 模式分析阶段
- **文本模式分析**: 
  - 平均文本长度: 27.8字符
  - 主要电压等级: 10kV (3671条), 20kV (110条), 35kV (85条)
  - 常见状态转换: 热备用↔运行, 冷备用↔运行, 检修↔冷备用

- **标注字段分析**:
  - 初始状态: 31种唯一值，主要为冷备用(1360)、热备用(1288)、运行(670)、检修(461)
  - 目标状态: 29种唯一值，分布类似初始状态
  - 操作类型: 主要为状态转换(3780)、拉合操作(82)
  - 设备名称: 2864种唯一值，高度多样化
  - 厂站名称: 50.7%为空值，1386种唯一值
  - 线路名称: 25.2%为空值，1704种唯一值

### 3. 规则提取阶段
基于模式分析，提取了以下关键规则：

- **状态转换规则**: `由(.{1,10})转(.{1,10})` 模式识别
- **设备名称规则**: 基于上下文的设备边界识别
- **厂站名称规则**: 以"站"、"所"结尾的实体识别
- **线路名称规则**: `电压等级+线路名` 模式识别

### 4. 算法开发阶段
开发了三个版本的抽取器：

1. **基础版本** (`auto_extractor.py`): 总体准确率75.83%
2. **改进版本** (`improved_extractor.py`): 总体准确率83.67%
3. **最终版本** (`final_extractor.py`): 集成最佳实践，提供完整API

## 最终成果

### 准确率表现
基于100条测试数据的评估结果：

| 字段 | 准确率 | 说明 |
|------|--------|------|
| 初始状态 | 98% | 表现优秀，"由...转..."模式识别准确 |
| 目标状态 | 93% | 表现良好，需要处理括号中的真实状态 |
| 操作类型 | 99% | 表现优秀，主要为状态转换 |
| 操作设备 | 38% | 挑战最大，设备名称边界不清晰 |
| 设备所属厂站 | 93% | 表现良好，厂站识别相对准确 |
| 设备所属线路 | 81% | 表现良好，线路模式识别有效 |
| **总体准确率** | **83.67%** | 整体表现良好 |

### 核心算法特点

1. **状态转换识别**: 使用正则表达式精确匹配"由...转..."模式
2. **设备名称抽取**: 基于上下文分析，移除线路/厂站前缀
3. **厂站识别**: 基于位置和后缀模式的双重验证
4. **线路识别**: 电压等级+名称的组合模式匹配
5. **错误处理**: 括号中真实状态的特殊处理

## 主要挑战与解决方案

### 1. 设备名称边界不清晰
**挑战**: 设备名称准确率仅38%，主要因为：
- 设备名称长度不固定
- 与线路名称、厂站名称边界模糊
- 包含复杂的修饰词和编号

**解决方案**:
- 基于上下文的分析方法
- 移除已识别的线路/厂站前缀
- 关键词匹配与长度优化结合

### 2. 括号中的真实状态
**挑战**: 部分文本的真实目标状态在括号中
**解决方案**: 特殊处理括号内容，提取真实状态

### 3. 数据质量不一致
**挑战**: 标注数据中存在空值和不一致性
**解决方案**: 数据清洗和容错处理

## 总结

本项目成功实现了基于人工标注数据的操作票信息自动抽取功能，总体准确率达到83.67%。通过深入的模式分析和迭代优化，建立了一套完整的抽取规则体系。虽然在设备名称抽取方面仍有改进空间，但整体效果已能满足实际应用需求。

该系统可以显著提高操作票信息处理的效率，减少人工录入工作量，为电力系统的智能化运维提供了有力支撑。
